# Qwen-2511 姿勢遷移 (傑尼引擎 Jojo 實戰版) - 專案總結與技術說明

## 專案概述
「Qwen-2511 姿勢遷移」是一個基於 Streamlit 構建的前端應用程式，旨在透過 ComfyUI 後端與 Qwen 視覺編輯模型，實現高精度的「人物姿勢轉移」。使用者只需上傳「人物原圖」與「姿勢參考圖」，系統即可在保留原圖人物特徵、服裝與臉部的同時，無縫替換為新的姿勢。

## 核心技術架構
1. **前端介面 (Streamlit)**：提供直覺的 1:1:2.5 視覺化版面配置，包含拖曳上傳、即時預覽、參數設定與結果展示。
2. **核心運算引擎 (ComfyUI)**：透過 REST API 與 WebSocket 與本機的 ComfyUI 進行通訊，動態注入 Jojo_workflow_api.json 工作流，即時回傳運算進度與生成節點狀態。
3. **AI 視覺擴充 (Nano Banana Pro / Gemini 2.5 Pro)**：整合 Google Generative AI API，提供「智能反推原圖特徵」功能，精準萃取原圖背景、光影與風格，自動生成對應的 Prompt，降低手動輸入的門檻。
4. **影像預處理 (Pillow)**：內建 create_padded_image_with_blur 邏輯，自動將上傳圖片等比例縮放，並使用模糊背景填補空白邊緣，確保送入模型的影像比例一致且邊界自然。

## 關鍵功能與流程
- **智能特徵反推**：點擊「智能反推原圖特徵」後，程式會將影像轉為 Base64 格式，送至 Gemini 2.5 Pro 模型，要求以 150 字內描述背景與氛圍，並自動替換至前端的提示詞輸入框。
- **雙重確認防呆機制 (Dialog)**：在執行耗時的 ComfyUI 運算前，會彈出居中視窗顯示當前提示詞，要求使用者點擊「是的，繼續下一步」才正式啟動，避免誤觸。
- **動態參數控制**：支援調整隨機種子 (Seed)、引導係數 (CFG)、運算步數 (Steps) 以及最長邊解析度 (Longest Edge Resolution)。
- **自動化存檔**：生成的結果會依照時間戳記自動命名並儲存於 Zeni_Results 資料夾。

## 未來擴充與維護備註
- API 金鑰已硬編碼/環境變數綁定。
- 連線逾時 (Timeout) 已調整為 45 秒，以適應 Gemini 大模型的深度視覺分析時間。
- Telegram Bot 若有 409 Conflict，需確保沒有其他設備 (如 Raspberry Pi) 佔用同一 Token。
