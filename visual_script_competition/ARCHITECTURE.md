# 視覺氛圍劇本比賽 (Visual Script Competition) - 思維架構與技術邏輯報告

## 1. 核心願景 (Core Vision)
本專案旨在創造一個基於 AI 代理人的「即時創意競技場」，將視覺藝術、文學創作與影視語言（分鏡、音效、溫度感）結合。透過不同特質的模型（導演、文學、裁判）在限定條件下競爭，產出具備高度「氛圍感」的短影音劇本。

## 2. 角色定義與能力分工
| 角色 | 預設模型 | 個性/技能標籤 | 職責 |
| :--- | :--- | :--- | :--- |
| **傑尼 (Zeni)** | `gemma4:31b-cloud` | 組織、企劃、公正裁決 | 出題、評分、維護總分榜、產出最終匯總報告 |
| **迪尼 (Dini)** | `glm-5:cloud` | 導演、攝影、分鏡、視覺化 | 以影像視角撰寫劇本，強調鏡頭運動與視覺氛圍 |
| **蝦蝦 (Xiaxia)** | `kimi-k2.5:cloud` | 創意、文學、細膩情感 | 以文字敘事切入，強調溫度感、氛圍與情感流動 |

## 3. 競賽邏輯流程 (Game Loop)
比賽採 **「交叉回答制」**，共 3 回合 (6 題)：
1. **出題階段** $\rightarrow$ 傑尼發布題目（包含季節/時間/場景暗示）。
2. **作答階段** $\rightarrow$ 參賽者依序提交劇本（優先順序每回合反轉）。
3. **裁決階段** $\rightarrow$ 傑尼根據評分矩陣計算分數。
4. **同步階段** $\rightarrow$ 伺服器更新 `visual_scoreboard.json` $\rightarrow$ GUI 介面即時反映。

## 4. 評分矩陣 (Scoring Matrix)
- **基礎分 (0-10)**：創意度、影像可視化程度。
- **必備要素 (Must-have)**：季節、時間(hh:mm)、地點、具體人物、食物溫度感。
- **加分項 (Bonus)**：
  - **三感滿足 (+3)**：同時具備【地點空間感、時間感、氣氛溫度感】。
  - **影視專業 (+2)**：包含【鏡頭運動描述】且標註【音效/配樂氛圍】。
- **懲罰項 (Penalty)**：
  - **超時 (180s)** $\rightarrow$ 該題 0 分。
  - **超字數 (>600字)** $\rightarrow$ 每多 50 字扣 1 分。

## 5. 技術架構 (Technical Stack)
- **後端**：Node.js (HTTP Server) $\rightarrow$ 管理狀態機與 JSON 檔案。
- **前端**：Vanilla HTML/JS $\rightarrow$ 輪詢/SSE 監控 API。
- **儲存**：JSON-based state management（無需資料庫，便於快速遷移與備份）。
- **啟動**：Windows `.bat` 批次檔 $\rightarrow$ 自動化部署流程。
